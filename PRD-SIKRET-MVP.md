# סיקרט — PRD (MVP)

**מוצר:** אפליקציה אחת — החלפת סודות (טקסט, ואופציונלית תמונה) עם זרים, בלי חשיפת זהות.  
**לא:** משחק, רשת חברתית, צ'אט, כספת כללית (ב-MVP).  
**כן:** מקום להתוודות או לשתף משהו אחד עם מישהו שלא מכיר אותך.

**משפט מוצר:** *סיקרט. חיים בלי סודות.*

**תאריך:** מאי 2026 | **גרסה:** MVP v1 (טקסט) · **v1.1** (תמונות + מנוי)

**המלצת השקה:** MVP = **טקסט בלבד**; תמונות ב-**v1.1** אחרי ייצוב תור/דיווחים (סיכון משפטי וחנויות גבוה יותר עם תמונות).

## 1. מטרה וקהל

| | |
|--|--|
| **בעיה** | אנשים רוצים לשחרר סוד/התוודות בלי מעקב, בלי פרופיל, בלי "למחוק אחר כך". |
| **פתרון** | מעטפה אחת ↔ מעטפה אחת (טקסט חובה; תמונה אופציונלית ב-v1.1), נעילה כפולה, פתיחה לפי כללים, אנונימיות. |
| **קהל** | מבוגרים (18+) שמחפשים שיתוף חד-פעמי, לא טיפול מקצועי. |
| **לא קהל** | צ'אט רציף, דייטינג, גלריות, שליחת וידאו/קול (לא ב-v1). |

## 2. עקרונות מוצר (לא מתפשרים)

1. **אין ביטול אחרי נעילה** — מה שננעל, נפתח רק לפי כללי הפתיחה (המותג).
2. **אנונימיות לזר** — הצד השני לא רואה שם, טלפון, פרופיל; רק תוכן המעטפה (אחרי הפתיחה).
3. **טקסט חובה בכל מעטפה** — תמונה אופציונלית (v1.1+), ללא וידאו/הקלטה ב-v1.
4. **החלפה אחת = מעטפה אחת** — אין צ'אט; "החלפה נוספת" רק אם **שניכם** מאשרים.
5. **לא צפייה לפני שהצד השני נעל** — בכל מצבי הפתיחה.
6. **זה לא טיפול** — אזהרה לפני שימוש ראשון / לפני נעילה.

## 3. זרימת משתמש (5 מסכים)

```
[1] כוונה (תגית חובה)
      ↓
[2] כתיבת מעטפה (טקסט, מינימום תווים)
      ↓
[3] איך תיפתח? (יחד | כשהצד השני נעול)  ← ברירת מחדל: יחד
      ↓
[4] תור → זיווג עם זר (תגית + ציון אמון פנימי)
      ↓
[5] נעילה → המתנה לצד השני → פתיחה → דירוג / דיווח / סיום
```

### מסך 1 — כוונה (תגית, לא תוכן)

- מתוודה / מחפש להאזין / סוד קטן / מכתב שלא נשלח (או רשימה קצרה דומה).
- משמש להתאמה בתור, לא לניתוח תוכן.

### מסך 2 — מעטפה

**MVP (v1) — טקסט בלבד**
- שדה טקסט.
- **מינימום:** 80–150 תווים (מומלץ 100).
- **מקסימום:** 1500 תווים.
- אין קבצים; חסימת URL בטקסט (אופציונלי).

**v1.1 — תמונה אופציונלית**
- טקסט **תמיד חובה** (גם עם תמונה — מינימום תווים נשאר).
- כפתור "הוסף תמונה" → בחירה מהגלריה / מצלמה (לא הדבקה ממקורות חיצוניים אם אפשר).
- **תמונה אחת** למעטפה; מקס גודל (למשל 5 MB); JPEG/PNG/WebP; רזולוציה מקס (למשל 2048px) אחרי דחיסה בצד לקוח.
- **לפני נעילה:** תצוגה מקדימה + אזהרה (פנים, מסמכים, עירום, QR = דיווח מהיר).
- **מטא-דאטה:** הסרת EXIF/GPS בשרת או בלקוח לפני העלאה.
- **התאמה בתור:** מי ששולח תמונה ↔ רק מי שבחר "מוכן לתמונה" (אופט-אין לשני הצדדים).

### מסך 3 — מצב פתיחה

| מצב | התנהגות |
|-----|----------|
| **יחד** (ברירת מחדל, מומלץ) | אף צד לא קורא עד ש**שניכם** במצב `locked`. |
| **כשהצד השני נעול** | אני נעול ומחכה; **לא** קורא עד שהזר נעל (אז נפתח לשנינו). |

**אסור ב-MVP:** "אני קורא לפני שהזר נעל".

### מסך 4 — תור וזיווג

- כניסה לתור לפי: תגית כוונה + ציון אמון פנימי.
- חדשים ↔ חדשים; ציון גבוה ↔ דומה.
- **יציאה מהתור לפני שנעלו שניהם** — מותר (עדיין לא נשלח כלום).
- **אחרי נעילה** — אין ביטול, אין עריכה.

### מסך 5 — אחרי פתיחה

- קריאת מעטפת הזר (פעם אחת; אופציונלי: סימון "נקרא").
- פעולות:
  - **נגע בי** → מעלה ציון אמון לשני הצדדים (אנונימי).
  - **לא כנה** → מוריד ציון; לא חושף זהות.
  - **דיווח** → ראה §6.
  - **סיום** / **החלפה נוספת?** (רק אם שניכם לוחצים).

## 4. אנונימיות וחשבון

| שכבה | מה נשמר | מה הזר רואה |
|--------|---------|-------------|
| חשבון (מערכת) | `user_id`, מכשיר, דיווחים, השעיות, מונה יומי | כלום |
| החלפה | `swap_id`, מעטפה מוצפנת, תגית, מצב פתיחה | מעטפה אחרי פתיחה בלבד |
| כינוי (אופציונלי) | `מעטפה-כחול-12` חד-פעמי | נמחק אחרי סגירת ההחלפה |

- אין פרופיל ציבורי, אין חיפוש משתמשים, אין רשימת חברים ב-MVP.
- **לפני נעילה — אזהרה:** "הזר יראה רק את מה שבמעטפה. אל תכלול שם, טלפון, רשתות חברתיות."

## 5. מגבלות שימוש ומנוי

### החלפות (טקסט)

| כלל | חינם | פרימיום |
|-----|------|---------|
| החלפות ביום | **3** | **8–10** (מגבלת קצב טכנית ~15) |
| מתאפס | חצות (שעון מקומי) | אותו דבר |
| מה נספר | רק החלפה **מושלמת** (נעילה כפולה + נפתחה) | אותו דבר |
| חשבון חדש (24–48 שע') | **1–2/יום** עד החלפה ראשונה מושלמת → 3 | אותו כלל |

### תמונות (v1.1)

| כלל | חינם | פרימיום |
|-----|------|---------|
| מעטפות **עם תמונה** ביום | **1** | **עד מכסת ההחלפות היומית** (למשל עד 3/10 לפי מנוי) |
| מונה נפרד | `image_envelopes_today` ≠ מונה טקסט | אותו מונה תמונה מורחב |
| טקסט בלי תמונה | לא פוגע במכסת התמונה | — |
| חשבון חדש | **0 תמונות** עד 1 החלפת טקסט מושלמת + ציון אמון מינימלי | או 0 עד 7 ימים |
| ציון אמון נמוך | תמונות **חסומות**; טקסט בלבד 0–1/יום | — |

**UX מומלץ:** אייקון מצלמה עם תגית "1 היום" / "פרימיום"; מעטפת עם תמונה מסומנת בתור כדי שהזר יודע לפני נעילה.

**מחיר (להגדרה):** מנוי חודשי — ישראל/גלובלי; לא חובה ל-PRD — רק עקרון: תמונה = ערך בתשלום, לא ליבת חינם.

## 6. אמון, ג'יבריש, דיווח והשעיה

### תהודה (1–5, גלוי) — טבעת בממשק

| אירוע | השפעה |
|--------|--------|
| "נגע בי" על המעטפה ששלחת | +0.35 |
| "לא כנה" | −0.6 |
| "דיווח" | −1.2 + השעיה |
| מוצג | בית + ליד מעטפת הזר + אחרי דירוג |
| התאמה | תהודה דומה בתור |

### ציון אמון פנימי (legacy, 0–100)

| אירוע | השפעה |
|--------|--------|
| החלפה הושלמה | + |
| "נגע בי" | ++ |
| "לא כנה" | −− |
| דיווח מאושר / חמור | השעיה |
| מעטפה מתחת למינימום | לא נשלחת |

### דיווח (אחרי פתיחה בלבד)

**סיבות:**
1. לא כנה / שטויות (ג'יבריש)
2. הטרדה / איום / תוכן קשה
3. ניסיון ליצירת קשר / מידע מזהה
4. **(v1.1) תמונה לא הולמת** — עירום, קטינים, אלימות, זיהוי (פנים/מסמך/מסך טלפון)

**הגנת מדווח שווא:** מקס 3 דיווחים/יום למדווח; מדווחים חוזרים מאבדים משקל.

### השעיה (בסגנון מדורג — טלגרם)

| שלב | טריגר (דוגמה) | תוצאה |
|-----|----------------|--------|
| 0 | דיווח "לא כנה" בודד | ירידת ציון / תור איטי |
| 1 | 3× "לא כנה" ב-30 יום | אזהרה + השעיה 24 שעות מהתור |
| 2 | דיווח חמור אחד (כולל תמונה פוגענית) | השעיה 7 ימים + בדיקה; **חסימת תמונות לצמיתות** |
| 3 | 5+ דיווחים חמורים / חוזר | 30 יום או חסימה |
| 4 | חוזר אחרי חסימה | חסימה לצמיתות |

- מסך מושעה + **ערעור** (מייל / טופס).
- השעיה **לא** מוחקת מעטפות שכבר נעלו (אין ביטול).

## 7. אבטחה ופרטיות

- **E2E** למעטפות טקסט (יעד); תמונות: לפחות הצפנה במנוחה + מפתח per-swap; E2E לתמונות אידיאלי.
- **תמונות (v1.1) — חובה לפני השקה:**
  - סריקת **CSAM / NCII** (PhotoDNA, Thorn, או ספק חנות) — דרישת חנויות וחוק במדינות רבות.
  - סריקת **NSFW/אלימות** לפני/אחרי העלאה (סיכון false positive — תהליך ערעור).
  - הסרת **EXIF**; חסימת PDF/וידאו disguised.
  - **Hash perceptual** — אותה תמונה מדווחת פעמיים → השעיה מהירה.
- שרת לא קורא תוכן לשיווק; מודרציה אוטומטית **רק** לתמונות (לא לטקסט E2E).
- **דיווח:** עותק תמונה לבדיקה רק במקרה דיווח, מוצפן, TTL 30 יום — ב-TOS במפורש.
- גיל מינימום **18+**; אימות גיל (אופציונלי v1.1) לפני שליחת תמונה.
- מצב הסוואה (אופציונלי): אייקון/neutral name בחנות.
- **אין** שמירת תמונה בגלריה אחרי סגירת מעטפה (אופציונלי: חסימת screenshot — Android limited, iOS limited — להצהיר "לא מבטיחים").

- וידאו, הקלטה, קבצים מרובים (v1)
- **תמונות (v1):** לא זמינות · **(v1.1):** עירום, קטינים, אלימות, צילום מסך מזהה, QR/טלפון בתמונה

**מוצג לפני נעילה (טקסט):**
> "מה שננעל לא ניתן לערוך, למחוק או לבטל. הזר יראה רק את מה שבמעטפה. זה לא שירות טיפולי."

**מוצג לפני נעילה (עם תמונה):**
> "התמונה תוצג לזר אנונימי אחד. אל תשלח פנים, מסמכים או תוכן מיני. מעטפת עם תמונה נספרת במכסה היומית."
